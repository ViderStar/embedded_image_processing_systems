# Лабораторная работа 1 — CDF 9/7 (лестничная схема, фиксированная запятой)

## Описание лестничной схемы CDF 9/7

Вейвлет CDF 9/7 (Cohen–Daubechies–Feauveau) используется в стандарте JPEG 2000.  
Преобразование реализуется через **лестничную схему** (lifting scheme) Sweldens без явной свёртки с длинными фильтрами.

### Шаги прямого 1-D преобразования

Входной сигнал `X[0..N-1]` разбивается на полифазные компоненты:

- **S** (нечётные отсчёты): `X[0], X[2], X[4], ...`
- **D** (чётные отсчёты): `X[1], X[3], X[5], ...`

Далее выполняются четыре лестничных шага и нормировка:

| Шаг | Операция | Коэффициент (рациональный) |
|-----|----------|----------------------------|
| 1 — predict | `D[i] += α·(S[i+1] + S[i])` | α = −3/2 |
| 2 — update  | `S[i] += β·(D[i] + D[i−1])` | β = −1/16 |
| 3 — predict | `D[i] += γ·(S[i+1] + S[i])` | γ = 4/5 |
| 4 — update  | `S[i] += δ·(D[i] + D[i−1])` | δ = 15/32 |
| 5 — scale   | `S[i] *= k`, `D[i] *= 1/k` | k = 4/5 |

На границах сигнала используется **полусимметричное** продолжение (symmetric extension).

### Фиксированная запятой (Q16)

Все отсчёты представляются в формате **Q16** (16 дробных бит).  
Умножение на рациональный коэффициент `num/den` выполняется целочисленно:

```
result = (value * num + den/2) / den
```

Это позволяет реализовать преобразование без операций с плавающей запятой, что важно для встраиваемых систем и FPGA.

### Обратное преобразование

Обратное преобразование выполняется в **обратном порядке**:
1. Денормировка (`S *= 1/k`, `D *= k`)
2. Шаги 4 → 3 → 2 → 1 с **отрицательными** коэффициентами

### Применение к изображению

1-D преобразование применяется **построчно** к изображению в оттенках серого.  
Для каждой строки получаются подполосы L (аппроксимация) и H (детали).

## Метрики

- **СКО (MSE)** — среднеквадратичная ошибка между исходным и восстановленным изображением
- **ПССШ (PSNR)** — пиковое отношение сигнал/шум, dB
